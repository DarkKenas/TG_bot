## Бот для поздравлений с днём рождения

Асинхронный Telegram‑бот на `aiogram`, который:

- собирает и хранит **данные пользователей** (ФИО, дата рождения);
- позволяет пользователям вести **личный вишлист** подарков;
- показывает **предстоящие и прошедшие дни рождения**;
- даёт возможность всем пользователям **предлагать варианты подарков** на дни рождения;
- предоставляет **панель коллектора** для просмотра предложений подарков;
- рассылает **уведомления о ближайших днях рождения**;
- имеет **админ‑панель** и отдельную роль **service_user** для управления администраторами;
- поддерживает **массовую рассылку и (там где возможно) закрепление сообщений** от админа.

---

## Стек и структура

- **Язык**: Python 3.11+
- **Фреймворк**: `aiogram`
- **БД**: PostgreSQL (`asyncpg` через SQLAlchemy)
- **ORM**: SQLAlchemy (async)
- **Миграции**: Alembic
- **Планировщик задач**: APScheduler
- **Конфигурация**: `pydantic-settings`

Запуск бота: файл `aiogram_run.py`.

Основные директории:

- `handlers/` — хендлеры сообщений и коллбеков;
- `keyboards/` — все inline / reply‑клавиатуры;
- `db_handler/` — модели, репозитории и работа с БД;
- `middlewares/` — DI, проверка регистрации и ролей;
- `scheduler_functions/` — периодические задачи (уведомления, запасной коллектор);
- `states/` — FSM‑состояния.

## Переменные окружения (.env)

Файл `.env` в корне проекта (читается `config.py`):

```env
TOKEN=ваш_telegram_bot_token

DB_HOST=localhost
DB_PORT=5432
DB_USER=postgres
DB_PASSWORD=your_password
DB_NAME=your_db_name

DEFAULT_SERVICE_USER_ID=123456789        # user_id сервисного пользователя
ADMIN_SECRET_CODE=секрет_для_получения_прав_админа
SERVICE_SECRET_CODE=секрет_для_получения_прав_service_user

BACKUP_COLLECTOR_USER_ID=123456789       # (опционально) запасной коллектор на 24 февраля
```

`pg_link` для SQLAlchemy формируется автоматически в `config.py`.

## Установка и запуск

```bash
python -m venv .venv
.venv\Scripts\activate  # Windows

pip install -r requirements.txt

# Применить миграции Alembic (если уже настроены)
alembic upgrade head

# Запуск бота
python aiogram_run.py
```

Планировщик APScheduler запускается внутри `aiogram_run.py`:

- ежедневные уведомления о ДР за **7 дней** и **1 день**;
- ежедневная очистка старых записей переводов/предложений после прошедших ДР;
- **24 февраля в 09:00** — автоматическое назначение запасного коллектора (`BACKUP_COLLECTOR_USER_ID`).

## Основные функции бота

- **Регистрация пользователя**:
  - ввод фамилии, имени, отчества, даты рождения;
  - после регистрации доступно главное меню.
- **Мои данные**:
  - просмотр текущих данных;
  - редактирование отдельных полей.
- **Мой вишлист**:
  - добавление желаний с текстом и (опциональной) ссылкой;
  - просмотр списка желаний;
  - редактирование текста/ссылки для конкретного желания;
  - удаление желаний.
- **Дни рождения**:
  - вывод списка предстоящих и прошедших ДР;
  - для предстоящих — визуальное разделение по месяцам;
  - просмотр вишлистов других пользователей из раздела "Дни рождения".
- **Предложения подарков**:
  - из уведомления о ДР можно нажать «🎁 Предложить вариант подарка»;
  - бот просит одним сообщением описать предложение и (опционально) добавить ссылку;
  - каждое предложение фиксируется в БД как запись в `transfers` (без денег, только `gift_text` и `gift_url`);
  - можно сделать **несколько предложений** для одного именинника.
- **Панель коллектора**:
  - доступна пользователям с ролью коллектора;
  - просмотр предложений подарков по именинникам (тексты + ссылки).
- **Уведомления о ДР**:
  - рассылка за **7 дней** и **1 день** до ДР всем пользователям (кроме именинника);
  - в тексте: дата, полное имя именинника, блок реквизитов активного коллектора;
  - для коллектора — отдельная информация по возрасту и юбилеям.
- **Роли и права**:
  - пользователь;
  - администратор;
  - service_user (может управлять списком администраторов);
  - коллектор (ответственный за сбор средств, данные для переводов используются в уведомлениях).
- **Массовая рассылка и закрепление**:
  - админ может разослать одно сообщение всем пользователям и попытаться его закрепить;
  - также можно массово снять закрепленные сообщения.

## Команды и скрытый функционал

Ниже перечислены все команды и важные действия, которые "спрятаны" в проекте.

### Пользовательские команды

- **`/start`**
  - Запуск бота и начало регистрации.
  - Если пользователь уже зарегистрирован, дальнейшая логика управляется `RegistrationMiddleware`.

### Кнопки главного меню

- **`👤 Мои данные`**
  - Показ текущих данных пользователя.
  - Кнопка «✏️ Редактировать данные» открывает меню редактирования полей.

- **`❤️ Мой вишлист`**
  - Показ своего вишлиста.
  - Под сообщением:
    - «➕ Добавить желание»;
    - «✏️ Редактировать вишлист» (если есть желания).

- **`📆 Дни рождения`**
  - Список предстоящих и прошедших ДР.
  - Кнопка «👥❤️ Вишлисты» под списком — просмотр вишлистов других пользователей по номеру.

- **`⚙️ Чат поддержки`**
  - Кнопка «💬 Написать» — прямой переход в чат с сервис‑пользователем (`tg://user?id=...`).

- **`⭕ Остановить ввод`**
  - Используется для выхода из режимов ввода (например, при просмотре вишлистов из раздела ДР).

### Получение ролей (скрытые команды)

- **`/get_admin`**
  - Начало получения прав администратора.
  - Бот запрашивает кодовую фразу (`ADMIN_SECRET_CODE`).

- **`/get_service_user`**
  - Начало получения прав service_user.
  - Бот запрашивает кодовую фразу (`SERVICE_SECRET_CODE`).

- После успешной проверки:
  - `/get_admin` — добавляет запись администратора и показывает в меню кнопку «🔐 Админ панель».
  - `/get_service_user` — назначает service_user и подсказывает использовать `/admin_list`.

### Команды service_user

- **`/admin_list`**
  - Доступно только для service_user.
  - Показ списка всех администраторов с нумерацией.
  - Ввод номера из списка — выбор админа для удаления прав.
  - Подтверждение через inline‑клавиатуру.

### Админ‑панель (кнопка «🔐 Админ панель»)

Доступна только администраторам, проверка — `RequireAdmin`.

Основные функции:

- Просмотр всех пользователей с нумерацией.
- **Назначение активного коллектора**:
  - выбор пользователя по номеру;
  - подтверждение;
  - обновление его меню (появляется «💰 Сбор панель»);
  - уведомление администраторов о новом активном коллекторе.
- **Удаление пользователя**:
  - выбор по номеру;
  - подтверждение удаления.

### Панель коллектора (кнопка «💰 Сбор панель»)

Доступна только пользователям с ролью коллектора.

Основное:

- Просмотр предложений подарков:
  - сгруппировано по именинникам;
  - показываются `gift_text` и, при наличии, кликабельные ссылки.

### Предложения подарков (из уведомлений)

- В уведомлении о ДР есть кнопка:

  - **`🎁 Предложить вариант подарка`**
    - При нажатии:
      - бот отправляет отдельное сообщение вида  
        `📅 02.02 день рождения отмечает ФИО`  
        `Опишите в одном сообщении ваши предложения по подарку:`
      - пользователь вводит текст;
      - далее бот предлагает добавить ссылку или нажать «Нет ссылки 🔗».
    - Каждое завершённое предложение создаёт запись в `transfers` с `gift_text` и `gift_url`.

### Админ‑команды рассылки

Доступны только администраторам (через `admin_router` + `RequireAdmin`):

- **`/pin_message`**
  - Бот просит ввести сообщение (текст/фото/документ).
  - Разсылает его всем зарегистрированным пользователям.
  - Пытается закрепить сообщение (в группах/каналах; в личных чатах закрепление через Bot API недоступно).
  - Возвращает админу отчёт:
    - сколько сообщений отправлено;
    - сколько ошибок;
    - сколько сообщений получилось закрепить.

- **`/unpin`**
  - Массово открепляет все закреплённые сообщения у всех пользователей (там, где API это позволяет).
  - Возвращает отчёт с количеством успешных и неудачных откреплений.

## Очистка БД от старых записей

Периодическая задача `clear_past_birthday_records`:

- запускается **каждый день в 00:00** (см. `aiogram_run.py`);
- находит пользователей, у которых день рождения уже прошёл в текущем году;
- удаляет:
  - все записи `Transfer` (включая предложения подарков) для этих именинников;
  - все записи `Greeting` (поздравления) для этих именинников.

Таким образом, предложения подарков **живут до конца текущего дня рождения**, а затем очищаются.

## Автоматический запасной коллектор

В `scheduler_functions/assign_backup_collector.py` реализовано:

- 24 февраля в 09:00 по таймзоне (`timezone` в `.env`) запускается задача;
- пытается назначить пользователя с `BACKUP_COLLECTOR_USER_ID` активным коллектором;
- при ошибках/отсутствии настроек отправляет предупреждения администраторам.

## Подготовка к деплою

Что уже сделано:

- Все клавиатуры вынесены в отдельные файлы в `keyboards/`.
- Все бизнес‑хендлеры разложены по модулям `handlers/`.
- Настроены middleware DI и проверки регистрации/ролей.
- Убраны тестовые и временные функции из основного кода (тестовая логика рассылки ДР — удалена).
- Настроена базовая структура миграций Alembic (`alembic/`).

Рекомендуемый чек‑лист перед продакшен‑запуском:

- **Заполнить `.env`** корректными значениями.
- **Создать и прогнать миграции Alembic** (если структура БД менялась):
  - `alembic revision -m "init"`
  - `alembic upgrade head`
- Убедиться, что в БД есть:
  - основной пользователь‑сервис (`DEFAULT_SERVICE_USER_ID`);
  - при необходимости — пользователь, который станет запасным коллектором (`BACKUP_COLLECTOR_USER_ID`).
- Проверить расписание задач APScheduler (локальное время vs. прод‑таймзона).
